# LeetCode Practice

A daily LeetCode practice repository for solving problems, tracking progress, and uploading solutions to GitHub.

## Purpose

This repository is used to:
- Practice one LeetCode problem each day
- Write clean solutions in a consistent format
- Track progress over time
- Share work publicly on GitHub

## Structure

Use a folder structure like this:

```text
Leetcode/
├── README.md
├── Day-001/
│   └── solution.py
├── Day-002/
│   └── solution.py
└── ...
```

You can also use different file names depending on the language:
- `solution.py`
- `solution.java`
- `solution.cpp`
- `solution.js`

## Daily Practice Template

For each problem, include:
- Problem name
- Problem link
- Date
- Approach / idea
- Time complexity
- Space complexity
- Solution code

Example:

```text
## Day 001 - Two Sum
- Date: 2026-06-30
- Problem: https://leetcode.com/problems/two-sum/
- Approach: Use a hash map to store seen values and their indices.
- Time Complexity: O(n)
- Space Complexity: O(n)
```

## Suggested Workflow

1. Pick one LeetCode problem each day.
2. Solve it on your own first.
3. Write the solution into a new folder or file.
4. Add a short explanation in the code or in a note.
5. Commit and push your work to GitHub.

## Example Commit Message

```text
git add .
git commit -m "Add Day 001 Two Sum solution"
git push
```

## Motivation

Consistency matters more than speed. Even solving one problem a day can build strong problem-solving habits over time.

## Note

Feel free to update this README as your practice routine grows.
